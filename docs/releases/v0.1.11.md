# Z Monitor Switcher v0.1.11

## English

### Highlights

- Fixed Ubuntu/Linux brightness and volume slider rendering so the thumb and
  track stay visually aligned while preserving the existing DDC write behavior.
- Improved Ubuntu window restore behavior after close-to-tray by avoiding the
  Linux always-on-top focus pulse and retrying focus after tray restore.
- Made Wayland Global Shortcuts Portal configuration recoverable: cancelled or
  empty system shortcut dialogs no longer clear existing bindings, and the same
  shortcut can be configured again immediately.
- Improved Linux launch-at-login reliability by verifying the generated
  autostart desktop entry and ensuring it launches with `--silent-start`.
- Autostart verification failures now surface in settings without blocking tray
  or hotkey initialization, and the optimistic toggle rolls back on failure.

### Notes

- Linux support targets Ubuntu 22.04 X11. Monitor compatibility still depends on
  DDC/CI support, `/dev/i2c-*` permissions, and the desktop environment.
- Ubuntu 22.04 X11 runtime verification for slider visuals, titlebar controls,
  native shortcuts, and launch-at-login should be performed with this release's
  GitHub Actions artifacts.
- Wayland shortcuts require a desktop environment that supports the Global
  Shortcuts Portal. X11 remains available for desktops that do not provide it.
- macOS builds target Apple Silicon and require macOS 12.0 or later.
- macOS builds are not signed or notarized yet; use right-click Open or the
  documented quarantine fallback if needed.
- Windows builds are unsigned and may show Microsoft SmartScreen.

## 简体中文

### 亮点

- 修复 Ubuntu/Linux 下亮度和音量滑块的视觉渲染问题，使滑块与滑条保持对齐，
  同时保留现有 DDC 写入行为。
- 改进 Ubuntu 关闭到托盘后的窗口恢复行为：Linux 下跳过 always-on-top 聚焦脉冲，
  并在托盘恢复后补一次聚焦，降低系统标题栏按钮卡住的概率。
- 优化 Wayland 全局快捷键 Portal 配置恢复体验：取消或未绑定系统快捷键时，
  不再清空已有绑定，并且可以立即重新点击同一个快捷键按钮配置。
- 改进 Linux 开机自启可靠性：启用后会验证生成的 autostart desktop entry，
  并确认启动命令包含 `--silent-start`。
- 自启验证失败会显示在设置里，但不会阻塞托盘或快捷键初始化；开关乐观状态也会在失败时回退。

### 注意事项

- Linux 支持目标是 Ubuntu 22.04 X11。显示器兼容性仍取决于 DDC/CI 支持、
  `/dev/i2c-*` 权限以及桌面环境。
- 本版本的 Ubuntu 22.04 X11 实机验证（滑块视觉、标题栏按钮、本机快捷键、开机自启）
  建议使用 GitHub Actions 构建产物进行测试。
- Wayland 快捷键需要桌面环境支持 Global Shortcuts Portal；不支持的桌面仍可使用 X11 会话。
- macOS 构建面向 Apple Silicon，要求 macOS 12.0 或更高版本。
- macOS 构建暂未签名/公证；如有需要，请使用右键打开或文档中的 quarantine
  备用命令。
- Windows 构建暂未签名，可能出现 Microsoft SmartScreen 提示。
