# Z Monitor Switcher v0.1.14

## English

### Highlights

- Improved native shortcut setup feedback when a shortcut cannot be registered
  because the key combination is already used by the system or another app.
- Kept Ubuntu 24.04 X11 shortcut behavior unchanged: successful shortcuts still
  use the native Tauri global-shortcut path and enter the existing input-switch
  and KVM confirmation flow.

### Notes

- Linux support targets Ubuntu 24.04 X11. Monitor compatibility still depends on
  DDC/CI support, `/dev/i2c-*` permissions, and the desktop environment.
- If a shortcut setup fails, choose a different key combination that is not
  reserved by the desktop, system settings, or another application.
- Wayland shortcuts remain best-effort through the desktop Global Shortcuts
  Portal. Desktops without Portal support should show a clear error; X11 remains
  the required supported shortcut baseline.
- macOS builds target Apple Silicon and require macOS 12.0 or later.
- macOS builds are not signed or notarized yet; use right-click Open or the
  documented quarantine fallback if needed.
- Windows builds are unsigned and may show Microsoft SmartScreen.

## 简体中文

### 亮点

- 优化本机快捷键设置失败时的提示：当组合键已被系统或其他应用占用时，提示用户更换组合键。
- 保持 Ubuntu 24.04 X11 快捷键业务逻辑不变：成功注册的快捷键仍走 Tauri
  原生全局快捷键路径，并进入既有输入源切换和 KVM 确认流程。

### 注意事项

- Linux 支持目标是 Ubuntu 24.04 X11。显示器兼容性仍取决于 DDC/CI 支持、
  `/dev/i2c-*` 权限以及桌面环境。
- 如果快捷键设置失败，请换用未被桌面环境、系统设置或其他应用占用的组合键。
- Wayland 快捷键继续通过桌面 Global Shortcuts Portal 做 best-effort 支持；
  不支持 Portal 的桌面应显示明确错误，X11 仍是必须支持的快捷键基线。
- macOS 构建面向 Apple Silicon，要求 macOS 12.0 或更高版本。
- macOS 构建暂未签名/公证；如有需要，请使用右键打开或文档中的 quarantine
  备用命令。
- Windows 构建暂未签名，可能出现 Microsoft SmartScreen 提示。
