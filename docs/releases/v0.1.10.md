# Z Monitor Switcher v0.1.10

## English

### Highlights

- Improved Linux tray behavior by keeping the tray menu to stable native actions
  for showing the main window and quitting the app, avoiding unreliable custom
  control popups on Linux desktops.
- Fixed normal app startup while preserving silent launch-at-login behavior: the
  main window is shown on regular launches and hidden only for silent starts.
- Polished the brightness and volume controls so vertical sliders keep a stable
  centered layout and use clearer accessibility labels.
- Removed misleading active-state highlighting from input-source buttons because
  many monitors do not reliably report the current VCP 0x60 input value.

### Notes

- Linux support targets Ubuntu 26.04. Monitor compatibility still depends on
  DDC/CI support, `/dev/i2c-*` permissions, and the desktop environment.
- Wayland shortcuts require a desktop environment that supports the Global
  Shortcuts Portal. X11 remains available for desktops that do not provide it.
- macOS builds target Apple Silicon and require macOS 12.0 or later.
- macOS builds are not signed or notarized yet; use right-click Open or the
  documented quarantine fallback if needed.
- Windows builds are unsigned and may show Microsoft SmartScreen.

## 简体中文

### 亮点

- 改进 Linux 托盘行为：Linux 托盘菜单仅保留稳定的本机操作（显示主窗口、退出应用），
  避免在不同桌面环境中不可靠的自定义快捷控制弹窗。
- 修复普通启动与静默自启的窗口行为：正常启动会显示主窗口，只有静默开机自启时才隐藏窗口。
- 优化亮度和音量控件布局，竖向滑块保持居中且尺寸稳定，并使用更清晰的无障碍标签。
- 移除输入源按钮的误导性当前状态高亮，因为很多显示器无法可靠返回 VCP 0x60 当前输入源。

### 注意事项

- Linux 支持目标是 Ubuntu 26.04。显示器兼容性仍取决于 DDC/CI 支持、
  `/dev/i2c-*` 权限以及桌面环境。
- Wayland 快捷键需要桌面环境支持 Global Shortcuts Portal；不支持的桌面仍可使用 X11 会话。
- macOS 构建面向 Apple Silicon，要求 macOS 12.0 或更高版本。
- macOS 构建暂未签名/公证；如有需要，请使用右键打开或文档中的 quarantine
  备用命令。
- Windows 构建暂未签名，可能出现 Microsoft SmartScreen 提示。
