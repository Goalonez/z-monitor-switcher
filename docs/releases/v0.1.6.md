# Z Monitor Switcher v0.1.6

## English

### Improvements

- Added a macOS-only keep-awake local control for clamshell and external-monitor
  KVM workflows. The setting is persisted and restored on launch, and uses a
  display-sleep prevention assertion while enabled.
- Added cleaning mode from the local controls area in both the main window and
  the menu-bar popup. It covers connected displays with black overlays and
  provides a centered exit button.
- Improved cleaning mode on macOS by raising overlays to screen-saver window
  level and swallowing non-Escape keyboard events when Accessibility permission
  allows the native event tap.
- Simplified the local system-volume row by removing the redundant text label
  and giving the slider more room.
- Avoid writing the same DDC input-source value again when the app already
  believes the monitor is on that source, reducing unnecessary USB-C
  renegotiation on clamshell Mac setups.

### Notes

- macOS builds target Apple Silicon and require macOS 12.0 or later.
- macOS builds are not signed or notarized yet; use right-click Open or the
  documented quarantine fallback if needed.
- Windows builds are unsigned and may show Microsoft SmartScreen.
- Cleaning mode's native keyboard suppression requires macOS Accessibility
  permission; without it, the overlay still captures focused keyboard events and
  app hotkeys are temporarily unregistered.

## 简体中文

### 改进

- 新增 macOS 专用“保持唤醒”本机控制，面向合盖 MacBook + 外接显示器 KVM
  工作流。开关状态会持久化并在启动时恢复，开启后使用防止闲置熄屏的系统断言。
- 新增“清洁模式”，入口位于主窗口和菜单栏弹窗的本机控制区域。开启后会用黑色覆盖层遮住已连接显示器，并在中央显示退出按钮。
- 改进 macOS 清洁模式：覆盖层提升到 screen-saver 窗口层级，并在系统辅助功能权限允许时吞掉 Escape 以外的键盘事件。
- 简化本机系统音量行，去掉重复的文字标签，让滑块获得更多空间。
- 当应用认为显示器已经处于目标输入源时，不再重复写入相同 DDC 输入源值，减少合盖 Mac 使用 USB-C 显示器时不必要的重新协商。

### 注意事项

- macOS 构建面向 Apple Silicon，要求 macOS 12.0 或更高版本。
- macOS 构建暂未签名/公证；如有需要，请使用右键打开或文档中的 quarantine
  备用命令。
- Windows 构建暂未签名，可能出现 Microsoft SmartScreen 提示。
- 清洁模式的原生键盘抑制需要 macOS 辅助功能权限；没有权限时，覆盖层仍会捕获焦点内键盘事件，并且应用全局快捷键会临时取消注册。
